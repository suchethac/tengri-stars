# tengri-stars — package design + bootstrap plan

## Context

New package `tengri-stars`: Bayesian, differentiable inference of single-star parameters
(Teff, logg, [Fe/H]; later distance, extinction, [α/Fe], RV) from photometry **and spectra**
(day-1 scope per user), targeting Gaia XP, LSST, DES, SkyMapper, CaHK. Modeled on the MAGIC
pipeline (Chiti 2026, arXiv:2605.26581; repo `ac8119/magic-processing-scratch`,
branch `refactor-rebuild`) whose inference stage is a *deterministic* cubic-spline
color-color interpolation with MIST isochrone logg priors — tengri-stars is its Bayesian,
gradient-based generalization, positioned against brutus/MINESweeper/Zhang-Green-Rix.
Joint development with Ani Chiti; starts on the user's GitHub account, later moves to the
tengri-project org. Starting grid: `TSLTE_combined_photometry.fits` (on Sherlock:
`/oak/stanford/orgs/kipac/users/achiti/grid/`), axes (teff, logg, feh) → per-filter AB mags
+ an `averaged` flag.

User decisions (2026-07-09): photometry+spectra day 1 · MAGIC parity as a validation
notebook, Bayesian fit is the product · distance/normalization pluggable ("do all in a
flexible way") · packaging: dependency (decided below after "help me decide").

## Decision: packaging — separate repo, tengri as a library dependency

- **Not a fork**: tengri main moves fast; a fork's copy of inference/observation/precompute
  rots immediately. **Not a monorepo**: couples release cycles, CI, and collaborator
  permissions (Ani works on stars, not galaxy internals). **Not a core split now**: big
  refactor before any science; can happen later *transparently* because tengri-stars imports
  stay `tengri.*` either way.
- Evidence the dependency route works: components registry accepts third-party subclasses
  (`src/tengri/components/sed_model_component.py` — `__init_subclass__` auto-registers into
  `_REGISTRY` from outside the package); `observation/`, `inference/`, `parameters/`,
  `protocols/` have no galaxy imports; alternate-model-class precedent exists
  (`PopulationSEDModel`, `SpatialSEDModel` on the SubModel protocol).
- Mechanics: `astro-tengri` is NOT on PyPI (name `tengri` is squatted by an unrelated
  project). tengri-stars pyproject dep: `astro-tengri @ git+https://github.com/suchethac/tengri.git`
  pinned to a tag/SHA. Publish `astro-tengri` to PyPI when tengri-stars first releases
  (PyPI rejects direct-URL deps in published packages).
- Upstream tengri PRs (small, filed as needed, none blocking M0–M2):
  1. `foreground` group: allow `ebmv_mw` as a Distribution (free parameter) — currently
     coerced to float at `src/tengri/parameters/groups.py:1348` (`_translate_foreground`).
  2. Lighter install: move `nifty8[re]`, `astroquery` to extras (backlog; galaxy users
     opt in).
  3. PyPI publication of astro-tengri.

## How the two packages work together

- **Runtime**: tengri-stars re-exports tengri types (`Uniform`, `Photometry`, `Observation`,
  …); users write `StarModel.build(grid=..., atmosphere={...}, distance={...},
  foreground={...})` — same grammar feel as `SEDModel.build`. `model.fit(..., method="nuts")`
  dispatches to tengri inference backends unchanged; `Posterior`/plotting come from tengri.
- **Ownership rule**: if a galaxy fit could also want it → tengri (filters, LSF,
  likelihoods, grid-interp, backends, analysis). If it mentions parallax/Teff/atmosphere
  grids → tengri-stars (loaders, star components, StarModel, distance treatments, MAGIC
  parity, survey glue).
- **Dev flow**: (1) pin astro-tengri to tag/SHA, deliberate bumps; optional nightly canary
  CI vs tengri main. (2) generic needs → upstream tengri PR → tag → bump (fittable
  ebmv_mw, RV shift, F99 law). (3) contract-test suite in tengri-stars exercising the
  tengri surfaces it uses (pattern: tengri's backend-conformance suite) so a failing bump
  names the seam. Stable boundary = tengri `protocols/` + public `tengri.*` re-exports.

## Architecture

```
            tengri-stars (new repo)                    tengri (dependency)
 ┌──────────────────────────────────────┐   ┌──────────────────────────────────────┐
 │ grids/    TSLTE phot FITS → mag LUT  │   │ utils/grid_interp: PreintegratedGrid,│
 │           spectral grid → F(λ) LUT   │──▶│   interp_nd_triweight, edges_for_grid│
 │ components/                          │   │ components/sed_model_component:      │
 │   AtmosphereGrid(SEDModelComponent)  │──▶│   registry, prior auto-discovery     │
 │   (photometry LUT path + spectrum    │   │ observation/: FILTER_REGISTRY,       │
 │    path; MW extinction reuses        │──▶│   compute_flux_density, apply_lsf,   │
 │    foreground laws; dilution)        │   │   calibration (Chebyshev), noise,    │
 │ model.py  StarModel (SubModel        │   │   NoiseModel, line_mask              │
 │   protocol; NOT SEDModel — ssp_data  │──▶│ forward/: orchestrator run_components│
 │   is hard-required there)            │   │   protocols/ (SEDComponent, Likeli-  │
 │ distance/ pluggable: dilution |      │   │   hood, ObservationModel, SubModel)  │
 │   parallax-prior | color-space |     │──▶│ inference/: InferenceContext, Fitter,│
 │   isochrone (phase 2)                │   │   MAP/NUTS/VI backends, Posterior    │
 │ parity/   MAGIC repro notebook       │   │ analysis/: corner, diagnostics       │
 └──────────────────────────────────────┘   └──────────────────────────────────────┘
```

- **StarModel, not SEDModel**: `SEDModel.build(cls, ssp_data, ...)` requires SSP data
  positionally (`src/tengri/forward/sed_model.py:5427`) and `_init_ssp` /
  `_validate_metallicity_bounds` / `_init_nebular` / `_init_multiwavelength` all consume it
  unconditionally. StarModel is a thin model class implementing the same protocol surface
  the inference layer consumes (reference implementations: `forward/population_sed_model.py`,
  `forward/spatial_model.py`), running star components through the shared orchestrator.
  Star physics still lives as `SEDModelComponent` subclasses — "physics within the forward
  model" as intended — so if tengri later grows an atmosphere-only SEDModel path, the
  components drop in unchanged.
- **Photometry channel**: fit-time prediction = single `interp_nd_triweight` lookup on the
  pre-integrated (teff, logg, feh) → per-filter-mag LUT (the SKIRTOR pattern,
  `components/agn/skirtor_precompute.py`), + extinction + dilution. No wavelength integral
  at fit time. Differentiable ⇒ MAP/NUTS/VI gradients free.
- **Spectroscopy channel (day 1)**: spectral grid (teff, logg, feh) → F(λ);
  interpolate spectrum → RV shift (new, small: wave/(1+v/c)) → `apply_lsf` (exists) →
  Chebyshev flux calibration (exists) → pixel likelihood (Gaussian/Student-t, exists).
  `velocity_broaden` (exists) covers vsini later.
- **Frame**: `redshift=Fixed(0)`; L→F dilution replaced by stellar normalization.
  Pluggable `distance` treatments selected at build time:
  `dilution` (one free nuisance ≈ DM + 5log(R/R☉) equivalent) · `parallax` (dilution with
  Gaia parallax prior) · `color` (distance-free color-space likelihood, MAGIC-style) ·
  `isochrone` (MIST link (Teff,logg,feh)→R; phase 2, needed for full MAGIC-parity logg
  prior anyway).
- **Extinction**: per-star fixed E(B-V) from SFD (MAGIC uses 0.65×SFD) via tengri
  `foreground` laws; free-E(B-V) option after upstream PR 1.

## Contracts to pin at the loader boundary

| Contract | Value / action |
|---|---|
| Grid axes | (teff [K], logg [dex], feh [dex]); `averaged` flag semantics — ask Ani |
| Grid payload | AB mags per filter (absolute/surface convention — pin at loader, convert once to a documented internal unit) |
| Filter convention | tengri default is photon-counting 1/λ (`FilterConvention.BESSELL`); verify what TSLTE grid used before any regeneration/mixing |
| Wavelengths | vacuum Å (tengri convention) — check spectral grids |
| Distance | parsecs via parallax; no cosmology anywhere |
| Spelling/style | American English, ruff config copied from tengri, same docstring standard |

## Milestones

- **M0 — scaffold**: `gh repo create suchethac/tengri-stars` (private), src layout
  (`src/tengri_stars/`), pyproject (BSD-3, git dep on astro-tengri pinned to SHA), ruff
  config + docstring standard copied from tengri, CI skeleton (lint + unit tests),
  README with the stack diagram. Data is NOT committed (download-on-demand like tengri).
- **M1 — grid ingestion**: FITS loader → LUT dataclass mirroring `PreintegratedGrid`
  (axes, edges via `edges_for_grid`, phot `(n_teff, n_logg, n_feh, n_filters)`), handling
  the `averaged` flag + non-rectangular coverage (mask/NaN policy); coverage plot;
  unit tests on a checked-in toy sub-grid. Get a local copy of the Sherlock FITS.
- **M2 — forward model**: `AtmosphereGrid` component (photometry + spectrum paths),
  MW extinction, dilution/distance plug-ins, `StarModel`; RV shift util.
  Extract the exact protocol surface `InferenceContext` requires by reading
  `population_sed_model.py` first (verify, don't assume).
- **M3 — inference wiring**: `InferenceContext.from_target(StarModel-fitter)` →
  MAP/NUTS/VI run unchanged; mock-star end-to-end recovery test (draw from grid, add
  noise, recover Teff/logg/[Fe/H] within errors) — the tengri mock-recovery pattern.
- **M4 — MAGIC parity notebook**: port the deterministic estimator (`getFeHs_v2`
  cubic-spline in (g−i, CaHK-index) per logg slice, isochrone logg prior, RGB/MS
  parallax classification) and compare star-by-star vs the Bayesian fit on the same grid —
  tengri's CIGALE/Prospector reproduction-notebook pattern.
- **M5 — applications**: Gaia XP synthetic photometry + LSST filter sets end-to-end;
  XP-coefficient-space fitting (Zhang-Green-Rix style) as phase 2.

## Verification

- M1: loader round-trip test (toy grid in repo); interpolation reproduces grid nodes
  exactly; gradient check (`jax.grad` finite-diff) through the LUT lookup.
- M2/M3: mock-star e2e recovery via the public predict API (verify e2e, not predict-state);
  fit < 1 min single star; OOM watchdog on any NUTS run (`LIMIT_GB=20
  scripts/run_with_oom_monitor.sh --` pattern from tengri).
- M4: star-by-star parity table vs MAGIC outputs; discrepancies attributed (grid-edge
  extrapolation, convention) before closing.
- Filter-convention audit: recompute a handful of grid magnitudes from raw TSLTE spectra
  through tengri's filter integration and compare to the FITS values.
